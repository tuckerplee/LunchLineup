# Compliance

Public SaaS privacy, security, and compliance documentation for LunchLineup.

## Files

- `README.md`: this compliance folder guide.
- `dpa-readiness.md`: public beta DPA request posture and paid GA legal blockers.
- `privacy-security.md`: public beta privacy and security commitments, current implementation evidence, and launch gaps.
- `soc2-readiness.md`: SOC 2 Type 1 readiness checklist.
- `subprocessors.md`: public beta subprocessor list and maintenance checklist.

## Current Focus

Use `privacy-security.md` before publishing public legal or trust pages. It defines the beta data categories, customer-facing commitments, and gaps that still need ownership before paid general availability.

Use `subprocessors.md` and `dpa-readiness.md` when updating `/subprocessors`, public contact routing, or customer-facing DPA wording.

Before treating public privacy, security, status, subprocessors, or DPA request copy as production-approved, record owner signoff for the monitored privacy, support, and DPA contact addresses configured through the public web environment.
