# Code Organization Hotspots

This is the production-readiness maintainability map for the rebuild checkout. It records source files that are large enough to hide mixed ownership, duplicated policy, or hard-to-test control flow.

Line counts were captured on 2026-07-09 from non-generated `apps/` and `packages/` source files, excluding tests, specs, build output, legacy `old/`, and generated artifacts. The hygiene suite now requires every non-generated source file over 500 lines to appear in this document.

## Status

- Fixed in this lane: README sibling inventories are covered by `tests/hygiene/repository-hygiene.test.mjs`, including tracked and untracked files. This document adds the missing maintainability routing for oversized source files.
- Deferred in this lane: application-code refactors. This slice owns docs, tests/hygiene, package/turbo config, and code-organization docs, so the source hotspots below are documented for follow-up rather than edited here.

## Hygiene Rule

The oversized-source threshold is intentionally conservative at 500 lines. A file can exceed it temporarily, but the public SaaS maintainability risk must be visible in this document before the migration hygiene suite passes.

## Recovery Debt

`schedules.controller.ts` and `lunch-breaks.service.ts` are temporarily recovered from the latest successful compiled API artifacts after a local editing-tool truncation. Their focused runtime suites and API typecheck pass, but the compiled-style TypeScript and relaxed class type aliases are maintenance debt. Reconstruct idiomatic typed source from the generated declarations and current behavioral tests before broad feature work in either module.
