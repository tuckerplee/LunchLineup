# Integration Tests

This folder contains service-backed smoke tests for the rebuild CI ephemeral stack plus a self-contained production module-wiring check. Service-backed tests require disposable Postgres, Redis, and RabbitMQ services and are intentionally separate from the fast migration hygiene suite.

## Command

Run from the repo root against disposable Postgres 16, Redis 7, and RabbitMQ 3 services. The test command does not start infrastructure, install Python dependencies, or provision the restricted application database role.

Set `MIGRATION_DATABASE_URL` to the disposable database owner URL, `DATABASE_URL` to the matching restricted application-role URL, `PLATFORM_ADMIN_DB_CONTEXT_SECRET` to the exact capability provisioned by the migration step, `REDIS_URL` to the disposable Redis URL, `RABBITMQ_URL` to a non-guest disposable broker URL, and `ENGINE_GRPC_URL` to an unused local `host:port`. The cancellation integration owner fails explicitly rather than skipping when either database URL or the capability is absent. The shared `DATABASE_URL` must not include Prisma-only query parameters such as `?schema=public`; the Python worker connects to the same URL through libpq. Install the pinned engine and worker requirements first. Before the test, run the same migration step as CI with `APP_DB_USER`, `APP_DB_PASSWORD`, `POSTGRES_USER`, `POSTGRES_PASSWORD`, `PLATFORM_ADMIN_DB_CONTEXT_SECRET`, `WEBHOOK_DELIVERY_ENCRYPTION_KEY_CURRENT`, and `DATA_TARGET_ENV=test` set:

```bash
python -m pip install -r apps/engine/requirements.txt -r apps/worker/requirements.txt
node scripts/apply-db-migrations.mjs
npm run test:integration
```

Tenant-scoped setup and verification statements use explicit transactions because `set_current_tenant` is transaction-local. The queue test generates Python clients from the checked-in `solver.proto` in a temporary directory, starts the established engine and worker entrypoints, uses unique queue/job identifiers, and removes processes, queues, generated files, and database rows in `finally` cleanup. The shift mutation tests require both URLs: their controller uses the restricted application role while fixture setup/readback and disposable audit cleanup use the migration owner.
