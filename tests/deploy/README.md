## Purpose

These tests keep server deploys tied to GitHub state, built Git-SHA image artifacts including the required backup image, digest-pinned base and third-party service images, isolated Compose project names, structured observability config validation with an optional real-tool validation contract, webhook replay worker startup, observable health surfaces, fail-fast runtime secrets, checked-in alert runbooks, scheduled encrypted offsite backups, backup freshness telemetry, retained-record purge telemetry, auth secret rotation, production runtime-env validation, retained launch-proof evidence, and machine-checkable restore proof instead of direct VM edits. CI DAST and load smoke must verify `release-manifest.json`, pull digest-pinned release images, and run Compose with `--no-build --pull never`; otherwise the smoke jobs can pass against source rebuilt or mutable-tag-pulled inside the runner instead of the artifact being deployed. Launch-proof tests also require command, exit-code, checksum, size, timestamp, source-SHA, and unique-URI metadata so a bare URL cannot satisfy production readiness.
